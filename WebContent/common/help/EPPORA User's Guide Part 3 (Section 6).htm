6     Interface (Tabs and Commands)

EPS has invested a significant amount of effort to make EPPORA’s Graphical User Interface (GUI) as intuitive as possible.  A guiding light in our GUI’s design is to minimize the number of “clicks” a user has to make to obtain his/her desired results.  Each page, after the Login page, contains a menu bar with four tabs containing commands for reports, projects, administration and help.  The Home page contains a project drop-down list along with a set of tasks for the user relative to project tasks he/she is supporting for the current allocation period.  It also contains any messages generated for the user from the EPPORA system.  For users with “Schedule” privileges, the Home page includes updated project schedule tasks from project team members for updated tasks for any project which the user is designated as the Project Manager or the Project Portfolio Manager.  For users with “Projects” privileges, the Home page includes a dashboard of approved projects for all divisions in project ranking order.  Section 6 describes the “Login” and “Home” pages along with the tabs and commands common to all EPPORA pages except the dialogues.

6.2    Home Page

The “Home Page” is where the major functions of EPPORA are selected.  Depending on the “Access Rights” of the individual that logged in, one or more of the tab commands in the “Home” page and other EPPORA pages may be dimmed (inaccessible).  For most tab commands it is necessary to select a project from the “Project Name” drop-down list.  The “EPPORA” system returns to the Home page after a user function is completed.  The Home page is invoked from the Login page.  Users can also select the “Home” tab and return to the Home page.

When a user logs into EPPORA, the user is presented with a set of assigned project tasks from the end-of-business allocation process along with any messages generated from the end-of-business processing.  The “Work Flow of Project Team Members” is displayed to users with “Schedule” privileges for those projects the user is either the Project Manager or the Project Portfolio Manager.  The “Approved Project Dashboard” is displayed for those users with “Projects” privileges for the projects within the divisions they support.

The “Work Flow” and “Work Flow of Project Team Members” supports the maintaining of project schedules.  Each day during the end-of-business processing, EPPORA automatically updates the “Work Flow,” displaying for all users their project task assignments.  Each project team member, at the end of their work updates the “Expended Hours” from their “Work Flow” display for each task they have worked on and if applicable the “Status” from their “Work Flow” and then clicks on the “Update” button.  This automatically sends to the relevant Project Manager and Project Portfolio Manager the updated information which becomes part of their “Work Flow of Project Team Members” display.  The Project Manager or Project Portfolio Manager then “Updates” (approves) or “Rejects” the updated information.  When “Update” is selected, the relevant project schedule is automatically updated.  If “Reject” is selected, then the project task is returned with an appropriate message.
